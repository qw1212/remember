# 安全架构文档

## 核心安全原则

### 1. 加密在Rust后端
- 核心加密逻辑不在JavaScript环境
- 前端仅负责UI交互
- 规避XSS、浏览器扩展等JS环境风险

### 2. 密钥硬件安全
- 密钥存储在系统Keychain/钥匙串
- 支持HSM/TPM硬件安全模块
- 密钥不可导出，不进入JS内存

### 3. 行业黄金标准算法
- **密钥派生**：Argon2id（内存硬约束，抗GPU暴力破解）
- **数据加密**：libsodium secretbox (XSalsa20-Poly1305)
- **认证加密**：AEAD，防篡改

## 数据加密流程

### 密码派生（首次设置主密码）

```
用户主密码
    │
    ▼
Argon2id（内存硬约束）
    │
    ├── 随机盐值（16字节）
    ├── 迭代次数（3）
    ├── 内存成本（64MB）
    │
    ▼
派生密钥（32字节）
    │
    ▼
存储到系统Keychain
```

### 数据加密（保存敏感数据）

```
明文数据
    │
    ▼
libsodium secretbox
    │
    ├── 随机Nonce（24字节）
    ├── 派生密钥（从Keychain读取）
    │
    ▼
密文 + Nonce
    │
    ▼
SQLite + SQLCipher存储
```

### 数据解密（读取敏感数据）

```
密文 + Nonce
    │
    ▼
从系统Keychain读取密钥
    │
    ▼
libsodium secretbox_open
    │
    ▼
明文数据（内存中，用后清除）
```

## 存储安全分级

### 最高安全级别（系统Keychain）

**适用数据：**
- 银行卡密码
- 其他密码
- 私钥
- 加密主密钥

**存储位置：**
- Windows: Credential Manager
- macOS: Keychain
- Linux: Secret Service

**安全特性：**
- 硬件安全模块支持（HSM/TPM）
- 操作系统级访问控制
- 密钥不可导出
- 加密存储在磁盘

### 高安全级别（SQLite + SQLCipher）

**适用数据：**
- 私密笔记
- 遗嘱内容
- 资产记录

**存储位置：**
- 本地文件系统
- SQLCipher加密数据库

**安全特性：**
- AES-256加密
- 页级加密
- 密钥由Rust侧管理

### 标准安全级别（SQLite）

**适用数据：**
- 习惯记录
- 知识库
- 梦想清单

**存储位置：**
- 本地文件系统
- 普通SQLite数据库

## 同步安全

### 端到端加密同步

```
本地数据
    │
    ▼
使用派生密钥加密
    │
    ▼
上传到用户云存储
    │
    ├── iCloud
    ├── OneDrive
    ├── Google Drive
    ├── Dropbox
    ├── WebDAV
    │
    ▼
其他设备下载
    │
    ▼
使用派生密钥解密
    │
    ▼
本地数据
```

### 同步安全原则

1. **主密码永不上传** - 主密码只在本地设备
2. **密钥不离开设备** - 派生密钥存储在系统Keychain
3. **只同步密文** - 云端只有加密后的数据
4. **用户完全控制** - 用户选择是否同步、使用哪个云存储

## 攻击防护

### 已防护的攻击

| 攻击类型 | 防护措施 |
|---------|---------|
| **XSS攻击** | 加密在Rust后端，JS无法访问密钥 |
| **浏览器扩展** | 桌面应用无浏览器扩展 |
| **内存溢出** | Rust编译时检查 |
| **暴力破解** | Argon2id内存硬约束 |
| **密钥泄露** | 密钥存储在系统Keychain，不可导出 |
| **数据篡改** | AEAD认证加密 |
| **中间人攻击** | 端到端加密 |

### 安全边界

**前端（Svelte）：**
- 仅负责UI交互
- 不处理加密逻辑
- 不存储密钥
- 不访问敏感数据明文

**后端（Rust）：**
- 核心加密逻辑
- 密钥管理
- 敏感数据存储
- 系统Keychain访问

## 最佳实践

### 开发者

1. **永远不要在前端处理加密**
2. **使用Tauri IPC机制通信**
3. **敏感数据用后清除**
4. **定期更新依赖库**
5. **进行安全审计**

### 用户

1. **使用强主密码**
2. **定期备份数据**
3. **不要分享主密码**
4. **启用系统锁屏**
5. **保持系统更新**

## 技术栈总结

| 组件 | 技术 | 安全特性 |
|------|------|---------|
| 前端 | Svelte + TypeScript | 编译时防护，攻击面小 |

| 数据存储 | SQLite + SQLCipher | 本地文件加密 |
