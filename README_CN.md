# remember（记忆）

> **一个私密的终身保险库，用于存储您的数字生活——不仅仅是密码和资产，更是您的人格、智慧、关系和记忆。**

**remember** 是您的个人数字档案库。它保存定义您的一切——从日常习惯和转瞬即逝的灵感，到您的人格特征、思维模式、情感反应，以及一生积累的智慧。它旨在帮助您整理、保存，并最终传承您的数字存在。

---

## 💡 为什么需要 remember？

我们的生活由无数信息碎片组成：账户密码、我们坚持的习惯、学到的知识、未来的梦想、重要的人际关系，以及最终我们必须面对的遗嘱和资产安排……这些碎片散落在各处。
**remember** 试图为它们提供一个统一、安全的家，可以陪伴您数十年。

**但 remember 不仅仅是管理数据：**
- **全面的数字遗产** – 为后代保存您的人格、智慧和记忆
- **超越资产** – 不仅是密码和财产，更是您的价值观、信念和人生哲学
- **面向未来** – 为潜在的AI集成整理您的数字存在
- **情感延续** – 让您所爱的人能够连接您的数字存在

***

## ✨ 核心功能

- 🔐 **数字凭证** – 账户、密码、私钥等，使用强本地加密存储
- 🧭 **习惯追踪器** – 每日打卡、连续天数统计、习惯养成曲线
- 📚 **知识库** – 学习笔记、书籍摘录、概念卡片
- 💭 **思想日记** – 日记、反思、感悟
- 🌟 **梦想清单** – 人生目标、愿望、进度跟踪
- 👥 **重要人际关系** – 家人、密友、关键联系人的联系和回忆
- 📜 **遗嘱与指示** – 数字遗嘱、身后安排、附件档案
- 💰 **资产** – 数字/实物资产、投资记录（加密存储）
- 🧪 **经验库** – 项目复盘、经验教训、人生经验

**所有数据本地优先，支持离线使用，并支持端到端加密。**

***

## 🔒 安全与隐私

- **零知识架构** – 平台无法访问您的任何明文数据；高度敏感数据使用军事级加密
- **完全离线能力** – 核心功能无需网络，数据完全由您控制
- **备份与恢复** – 支持加密导出/导入，防止单点故障导致数据丢失
- **遗产交接** – 预设"数字遗产联系人"和时间触发机制（例如长时间不活动），确保必要时关键信息能传递到可信赖的人手中

> 这是您自己的"人生数据库"，而非云服务提供商的数据矿藏。

***

## 🎯 愿景与边界

### 核心意义

**remember** 代表了数字时代对数据主权的回归。它不仅仅是一个工具，更是一种理念：您的数字生活应该像您的物理生活一样私密和可控。这个项目解决了三个关键需求：

1. **数据碎片化** – 将分散的个人信息整合到一个安全的位置
2. **隐私侵蚀** – 为依赖云服务并从您的数据中获利的服务提供真正的替代方案
3. **数字遗产** – 确保您的数字存在能够被妥善管理、保存和传承

### 扩展可能性

项目可以在以下方向谨慎发展：

- **家庭共享** – 安全的密码共享和家庭记忆库
- **时间胶囊** – 在预定的未来日期解锁的内容
- **生物识别集成** – 指纹和面部识别以增强安全性
- **多设备同步** – 跨设备访问，保持加密安全
- **法律集成** – 与数字遗产规划服务合作
- **选择性AI集成** – 用户选择哪些数据与AI共享（如习惯、知识），同时保持敏感数据（密码、私密笔记）加密隐藏。支持自定义AI API以完全控制。

### 个人数字档案系统（未来愿景）

**remember** 旨在成为全面的个人数字档案库：

- **人格建模** – 捕获您的人格特征、思维模式、情感反应和沟通风格
- **关系网络** – 映射您的人际关系、互动历史和情感连接
- **记忆时间线** – 保存重要人生事件、成长轨迹和关键记忆
- **知识与智慧** – 存储您的专业知识、人生经验、价值观和决策模式
- **创造性输出** – 记录您的写作风格、创意偏好和艺术审美
- **AI交互** – 使未来AI能够理解您的沟通风格和价值判断

### 明确边界

为了维护其核心原则，**remember** 永远不会包含：

- ❌ **社交网络功能** – 支持有限的共享（家庭密码），但这不是社交平台
- ❌ **数据商业化** – 没有广告，没有分析销售，没有第三方数据共享
- ❌ **强制云依赖** – 本地优先，可选加密云备份
- ❌ **不必要的复杂性** – 没有区块链、加密货币或过度工程化的解决方案

> **remember** 是您的数字圣殿——私密、安全、完全属于您。

## 🛠️ 技术栈

- **前端框架**：React + TypeScript
- **构建工具**：Vite
- **本地数据库**：IndexedDB + Dexie.js（浏览器原生）
- **加密**：Web Crypto API（AES-256-GCM）
- **UI 框架**：Tailwind CSS + shadcn/ui
- **状态管理**：React Context + Zustand
- **桌面应用**：Tauri（推荐）/ Electron
- **后端**：第一阶段无需后端（纯静态托管）
- **部署**：静态托管（Vercel/Netlify）– **完全免费**

***

## 🔐 安全防护

### 数据加密

| 层级 | 技术 | 说明 |
|------|------|------|
| 密钥派生 | PBKDF2 | 100,000次迭代，SHA-256 |
| 数据加密 | AES-256-GCM | 军事级加密标准 |
| 存储 | IndexedDB | 本地加密存储 |

### 桌面应用安全优势

使用 **Tauri** 打包桌面应用提供更高安全性：

- ✓ 数据存储在本地文件系统，非浏览器环境
- ✓ 可使用操作系统安全存储（Windows Credential Manager / macOS Keychain）
- ✓ 无XSS攻击风险
- ✓ 无浏览器扩展风险
- ✓ 原生系统级安全保护

### 安全建议

| 部署方式 | 安全级别 | 适合存储的数据 |
|---------|---------|--------------|
| **桌面应用（Tauri）** | 高 | ✓ 密码、财务、机密数据 |
| PWA | 中 | 一般个人数据 |
| 纯Web | 低-中 | 一般个人数据 |

> **重要提示**：存储密码、财务等高度敏感数据时，强烈建议使用桌面应用版本。

## 📱 产品架构：本地优先 + 可选同步

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    用户主密码                             │
│               （永远不离开本地设备）                        │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ▼               ▼               ▼
    ┌─────────┐     ┌─────────┐     ┌─────────┐
    │ 电脑端  │     │ 手机端  │     │  平板   │
    │ 桌面应用 │     │  移动App │     │  App   │
    │ (主要)  │     │ (辅助)  │     │ (辅助) │
    └─────────┘     └─────────┘     └─────────┘
          │               │               │
          └───────────────┼───────────────┘
                          ▼
              ┌─────────────────────┐
              │   加密数据同步       │
              │  (用户自己的云存储)  │
              │  iCloud/OneDrive等  │
              └─────────────────────┘
```

### 各端职责

| 平台 | 角色 | 主要功能 |
|------|------|---------|
| **电脑端** | 主要 | 完整功能管理、大量数据输入、复杂操作、安全级别最高 |
| **手机端** | 辅助 | 快速查看密码、生物识别解锁、临时记录、拍照存档 |
| **平板** | 辅助 | 中等屏幕体验、介于电脑和手机之间 |

### 同步安全设计

| 原则 | 实现方式 |
|------|---------|
| **数据加密** | AES-256-GCM，本地加密后才同步 |
| **主密码** | 永远不离开本地设备，不上传云端 |
| **同步内容** | 只同步加密后的密文，不含明文 |
| **用户控制** | 用户选择是否同步、使用哪个云存储 |

### 支持的云存储（可选）

- iCloud（Apple设备）
- OneDrive（Microsoft）
- Google Drive
- Dropbox
- 自建WebDAV

> **核心原则**：数据始终属于用户，云存储仅作为加密数据的传输通道。

### 第二阶段：功能扩展（4-6周）

- 习惯追踪功能
- 知识库功能

- **关系网络** ✓ – 映射人际关系、互动历史、情感连接
